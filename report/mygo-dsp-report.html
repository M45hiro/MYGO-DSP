第一章 项目概述

1.1 什么是 MYGO-DSP？

MYGO-DSP 是一个数字信号处理（Digital Signal Processing，简称 DSP）软件平台。如果你完全没有听说过数字信号处理，别担心——我们会从最基础的概念开始解释。

简单来说，数字信号处理就是用计算机来处理"信号"。什么是信号呢？声音就是信号——你说话时声带振动产生的声音，被手机麦克风录下来，转换成数字信号，然后经过各种处理（比如降噪、美声、加特效），最后从扬声器播放出来。这一整条链路的核心技术就是数字信号处理。

MYGO-DSP 的全称是 "Modular Yielding Graphical Operators"，中文可译为"模块化高效图形运算器"。其中每个单词的含义是：Modular（模块化的）意味着系统的各个功能（波形发生器、滤波器等）都是独立的模块，你可以像搭积木一样自由组合它们。Yielding（高效产出的）意味着系统经过性能优化，能够高效地产生和处理音频信号。Graphical（图形化的）意味着系统提供了一个美观的图形用户界面，让你可以用鼠标点击和拖拽来操作，不需要写代码。Operators（运算器）意味着系统中的每个功能模块本质上都是一个"运算器"——输入信号进去，经过数学运算，输出处理后的信号。

MYGO-DSP 项目包含以下几个主要部分：C++ DSP 引擎是用 C++ 编写的核心计算模块，负责所有数学运算——生成波形、设计滤波器、处理音频数据等。这部分是系统的"大脑"，运行在后台，没有图形界面。Electron 桌面应用是用 Web 技术（HTML、CSS、JavaScript）构建的图形界面，看起来像一个原生的桌面软件。你可以在上面点击按钮、拖动滑块、查看图表。IPC 通信协议是 C++ 引擎和界面程序之间通过 JSON 格式的消息进行通信。IPC 是 Inter-Process Communication（进程间通信）的缩写。

1.2 M.Y.G.O 四个字母的含义

M 代表 Modular（模块化）：在软件工程中，模块化是一种将系统分解为独立、可互换的模块的设计方法。每个模块封装了特定的功能，并通过明确定义的接口与其他模块交互。MYGO-DSP 的模块化体现在：波形发生器、FIR 滤波器、IIR 滤波器、智能设计器等都是独立的模块，它们通过 IOperator 接口统一管理，可以独立开发、测试和替换。

Y 代表 Yielding（高效产出）：在数字信号处理中，实时性是一个关键指标。一个 44.1kHz 采样率的音频系统每秒需要处理 44,100 个采样点，每个采样点的处理时间只有约 22.7 微秒。MYGO-DSP 通过高效的算法实现（如环形缓冲区、级联双二阶结构、查表法三角函数等）和 C++ 的高性能特性（如移动语义、内联函数、编译器优化），在性能测试中达到了每秒处理超过 4500 万个采样点的成绩。

G 代表 Graphical（图形化）：图形用户界面（GUI）使得复杂的 DSP 操作变得直观易懂。用户不需要编写任何代码，只需通过鼠标点击和拖拽就可以：选择波形类型、调节频率和幅度参数、添加滤波器、查看实时的时域和频域图表。

O 代表 Operators（运算器）：在数学和计算机科学中，算子（Operator）是一个函数，它将一个或多个输入映射到输出。在 DSP 中，算子就是对信号进行某种变换的数学操作。MYGO-DSP 中的每个功能模块本质上都是一个算子，它们可以串联、并联、组合，实现复杂的信号处理链路。

1.3 数字信号处理简介（面向零基础读者）

什么是信号？

信号就是"携带信息的物理量随时间的变化"。举例来说：你说话时，空气的振动是声学信号；麦克风把声音变成电压变化，这是电信号；手机把电压变化变成一串数字，这是数字信号。数字信号由一系列采样点（samples）组成。每个采样点是一个数字，代表该时刻信号的"高度"（幅度）。

数学上，一个离散时间信号可以表示为 x[n]，其中 n 是整数索引（第 n 个采样点）。例如，一个正弦波信号可以写成：

x[n] = A \cdot \sin(2\pi f n / fs + \phi)

其中 A 是幅度，f 是频率，fs 是采样率，\phi 是初始相位。

什么是采样率？

采样率（Sample Rate）表示每秒钟采集多少个采样点，单位是 Hz（赫兹）。CD 音质的采样率是 44,100 Hz，即每秒采集 44,100 个点。专业音频设备常用 48,000 Hz 或 96,000 Hz。采样率越高，能还原的信号频率越高。根据奈奎斯特采样定理（Nyquist Theorem）：要无失真地还原一个信号，采样率必须至少是该信号最高频率的两倍：

fs \geq 2 f\text{max}

例如，人耳能听到的最高频率约为 20,000 Hz，所以理论上 40,000 Hz 以上的采样率就足够了。

什么是频率？

频率（Frequency）表示信号每秒振动的次数，单位也是 Hz。中央 C（中音多）的频率约为 261.6 Hz。标准音 A4 的频率是 440 Hz。人耳能听到的频率范围大约是 20 Hz 到 20,000 Hz。频率与周期的关系是：

f = 1⁄T

什么是滤波器？

滤波器（Filter）是一种"选择性通过"的装置。低通滤波器让低频信号通过，阻止高频信号。高通滤波器让高频信号通过，阻止低频信号。带通滤波器只让某个频率范围内的信号通过。带阻滤波器阻止某个频率范围内的信号通过。

滤波器的性能由几个关键参数描述：

  • 截止频率：增益下降 3dB 的频率点
  • 通带：信号被允许通过的频率范围
  • 阻带：信号被衰减的频率范围
  • 过渡带：通带到阻带之间的过渡区域
  • 纹波：通带或阻带内的增益波动范围

什么是波形？

波形（Waveform）是信号形状的图形表示。常见的波形类型包括：正弦波是最纯净的波形，只有一个频率成分。方波在正负最大值之间瞬间切换，含有丰富的奇次谐波。三角波类似正弦波但更"尖"，谐波含量比方波少。锯齿波含有所有整次谐波。白噪声是随机信号，在所有频率上有相同的能量。粉红噪声每倍频程能量相同，听起来比白噪声更"柔和"。

1.4 技术架构概述

MYGO-DSP 采用前后端分离的三层架构。第一层是 C++ DSP 引擎（后端计算层），负责所有复杂的数学运算。第二层是 Electron 主进程（中间桥接层），负责管理窗口和进程间通信。第三层是 React 前端（展示层），提供图形用户界面。

数据流向：用户在界面上配置参数，React 前端调用 window.dsp API，通过 IPC 发送到主进程，主进程构造 JSON 写入引擎 stdin，引擎计算后通过 stdout 返回结果，最终渲染到图表上。整个往返时间通常只有 1-5 毫秒。

1.5 完整功能列表

波形发生器

MYGO-DSP 提供了 7 种波形发生器：正弦波、方波、三角波、锯齿波、白噪声、粉红噪声、脉冲波。每个波形可以独立控制频率（1-96,000 Hz）、幅度（0-1）、相位（0-360°）、占空比（0-100%）。

FIR 滤波器

支持五种窗函数：矩形窗、汉明窗、汉宁窗、布莱克曼窗、凯泽窗。四种滤波器类型：低通、高通、带通、带阻。阶数范围 1-200。

IIR 滤波器

支持六种模拟原型：巴特沃斯、切比雪夫 I/II 型、椭圆、贝塞尔、勒让德。九种 RBJ 双二阶：低通、高通、带通（两种）、带阻、低架、高架、带架、全通。

智能滤波器设计

根据技术指标自动选择最佳滤波器原型和阶数，在 IIR 不可行时自动回退到 FIR 凯泽窗设计。

实时可视化

时域波形图、FFT 频谱图、零极点图。

第四章 C++ DSP 核心

4.1 C++ 基础概念（面向零基础读者）

什么是变量？

变量就像有标签的盒子。C++ 声明格式：类型 变量名 = 值;。例如 int、double、float、char、bool。

什么是函数？

函数是一段有名字的代码块，接收输入（参数），返回结果。格式：返回类型 函数名(参数) { 代码; }。

double square(double x) {
    return x * x;
}

什么是类？

类把数据和操作数据的方法打包在一起。包含成员变量（属性）和成员函数（方法）。

class Dog {
public:
    std::string name;
    void bark() { std::cout << name << " says: Woof!"; }
};

什么是构造函数？

构造函数在创建对象时自动调用，用于初始化成员变量。名称和类名相同，无返回值。

什么是继承？

子类从父类继承属性和方法。WaveformGenerator 等继承自 IOperator。

什么是虚函数？

virtual 关键字实现多态。纯虚函数（= 0）在抽象类中声明，子类必须实现。

class IOperator {
public:
    virtual void process(const double* input, double* output, sizet numSamples) = 0;
};

什么是智能指针？

std::uniqueptr 自动管理对象生命周期，避免内存泄漏。

什么是 std::vector？

动态数组，长度可运行时改变，内存连续，支持 pushback、size、[] 等操作。

什么是 std::unorderedmap？

哈希表，存储键值对。Engine 用其按 ID 快速查找波形和滤波器。

什么是互斥锁？

std::mutex 保护共享数据，防止多线程竞态条件。std::lockguard 是 RAII 风格的锁管理。

4.2 DSPMath.h — 数学常量定义

#pragma once
#ifndef DSPMATHH
#define DSPMATHH
#ifndef MPI
#define MPI 3.14159265358979323846
#endif

}

在信号处理中，\pi 无处不在。正弦函数、滤波器设计、傅里叶变换都离不开 \pi。

4.3 Buffer.h — 环形缓冲区

环形缓冲区（Ring Buffer）是一种高效的数据结构，用读/写指针管理数据。写指针指向写入位置，读指针指向读取位置。到达末尾时自动回卷。

push 函数写入数据，pop 函数读取数据。使用模板类 template 支持任意类型。

4.4 IOperator.h — 算子抽象接口

IOperator 是整个 DSP 核心的基石。定义四个纯虚函数：

  • process：处理音频数据
  • reset：重置内部状态
  • isBypassed：查询旁路状态
  • setBypass：设置旁路

使用了策略模式（Strategy Pattern）和接口隔离原则。

4.5 WaveformGenerator — 波形发生器

枚举 WaveformType 定义了 7 种波形。参数结构体 WaveformParams 包含频率、幅度、相位、占空比等。

核心算法：相位累加器

相位累加器维护 \phi in [0, 1)，每步增加 f/fs：

\phi[n] = (\phi[n-1] + f/fs) \bmod 1

正弦波：\sin(2\pi t + \phi)。方波的傅里叶级数：

x(t) = 4A⁄\pi \sumk=1,3,5,\ldots \sin(2\pi k f t)⁄k

白噪声使用均匀分布随机数，粉红噪声通过 6 个一阶低通滤波器将白噪声转换为 1/f 噪声。

4.6 FIRFilter — FIR 滤波器

FIR 是 Finite Impulse Response。特点：有限脉冲响应、绝对稳定、线性相位、阶数较高。

输出是输入与脉冲响应的卷积：

y[n] = \sumk=0N-1 h[k] \cdot x[n-k]

FIR 使用环形缓冲区管理延迟线，将数据移动复杂度从 O(N) 降低到 O(1)。

窗函数法设计

第一步：计算理想脉冲响应

h\text{ideal}[n] = \sin(\omegac (n-M))⁄\pi (n-M)

第二步：加窗。五种窗函数：矩形、汉明、汉宁、布莱克曼、凯泽。第三步：归一化。

4.7 IIRFilter — IIR 滤波器

IIR 是 Infinite Impulse Response。特点：无限脉冲响应、效率高（阶数低）、可能不稳定、非线性相位。

y[n] = \sumk=0M bk x[n-k] - \sumk=1N ak y[n-k]

双二阶节级联

H(z) = \prodk=1K b0 + b1 z-1 + b2 z-2⁄1 + a1 z-1 + a2 z-2

级联提高了数值稳定性。模拟原型包括巴特沃斯、切比雪夫 I/II、椭圆、贝塞尔、勒让德。

巴特沃斯：

|H(j\omega)|2 = 1⁄1 + (\omega/\omegac)2N

4.8 AutoFilterDesigner — 智能设计器

根据规格自动选择最优滤波器。阶数估算公式：

  • 巴特沃斯：N = \lceil \log10((10As/10 - 1)/(10Ap/10 - 1))⁄2\log10(\omegas/\omegap) \rceil
  • 凯泽窗 FIR：N = \lceil As - 7.95⁄14.36 \cdot \Delta f \rceil

4.9 Engine — 引擎调度器

采用单例模式（Singleton Pattern）。C++11 线程安全的静态局部变量初始化。Engine 管理波形和滤波器的生命周期，processAudio 实现完整的音频处理管线。

4.10 main.cpp — IPC 循环

程序入口。手工实现 JSON 解析（extractString、extractNumber、extractInt）。支持 13 个命令：init、addWave、removeWave、updateWave、addFilterFIR、addFilterIIR、removeFilter、updateFilter、setBypass、process、autoDesign、getCounts、exit。

第九章 算法深度解析

9.1 傅里叶分析基础

任何周期信号可分解为正弦波的和。傅里叶级数：

x(t) = a0 + \sumn=1\infty (an \cos(2\pi n f0 t) + bn \sin(2\pi n f0 t))

离散傅里叶变换（DFT）：

X[k] = \sumn=0N-1 x[n] e-j2\pi kn/N

FFT 将复杂度从 O(N2) 降低到 O(N log N)。256 点 FFT 只需 1024 个蝶形运算。

9.2 窗函数理论

窗函数在主瓣宽度和旁瓣衰减之间权衡：矩形窗（4pi/N, -13dB），汉明窗（8pi/N, -43dB），汉宁窗（8pi/N, -32dB），布莱克曼窗（12pi/N, -58dB），凯泽窗（beta 可调）。

9.3 FIR 窗函数法设计

理想低通脉冲响应：

hd[n] = \sin(\omegac n)⁄\pi n

加窗：h[n] = hd[n] \cdot w[n]。归一化：h\text{norm}[n] = h[n] / \sum h[k]。

9.4 IIR 模拟原型

巴特沃斯极点：

pk = \exp\left(j\left(\pi⁄2 + (2k+1)\pi⁄2N\right)\right)

切比雪夫 I 型极点分布在椭圆上。椭圆滤波器最优（通带和阻带内等波纹）。贝塞尔多项式递推：

B0(s) = 1,\quad B1(s) = s + 1,\quad Bk(s) = (2k-1) Bk-1(s) + s2 Bk-2(s)

9.5 双线性变换推导

s = 2⁄T \cdot 1 - z-1⁄1 + z-1

频率翘曲：\Omega = 2⁄T \tan\left(\omega T⁄2\right)。预翘曲补偿非线性映射。

9.6 零极点分析

H(z) = K \cdot \prod (z - zk)⁄\prod (z - pk)

系统稳定的条件是所有极点在单位圆内（|pk| < 1）。

9.7 频率响应计算

H(ej\omega) = H(z)|z=ej\omega

幅度响应（dB）：20 \log10(|H(ej\omega)|)。群延迟：\tau(\omega) = -d\arg(H)⁄d\omega。

9.8 椭圆积分（AGM 算法）

K(k) = \int0\pi/2 d\theta⁄\sqrt{1 - k2 \sin2 \theta}

AGM 算法：初始化 a0=1, b0=sqrt(1-k2)，迭代 a=(a+b)/2, b=sqrt(a*b)，收敛后 K(k)=pi/(2*a)。收敛极快，4-5 次迭代即达双精度。

9.9 设计实例对比

规格：通带 1kHz，阻带 2kHz，纹波 0.5dB，衰减 40dB，采样率 44.1kHz。

巴特沃斯：N = \lceil \log10((104-1)/(100.05-1)) / (2\log102) \rceil = \lceil 8.16 \rceil = 9 阶

切比雪夫 I 型：N = \lceil 4.82 \rceil = 5 阶

椭圆：仅需 3-4 阶。椭圆滤波器效率最高。

附录 A：扩展算法推导

A.1 离散傅里叶变换的完整推导

DFT：

X[k] = \sumn=0N-1 x[n] \exp\left(-j2\pi kn⁄N\right)

逆变换 IDFT：

x[n] = 1⁄N \sumk=0N-1 X[k] \exp\left(j2\pi kn⁄N\right)

DFT 复杂度 O(N2)，FFT 复杂度 O(N log N)。Cooley-Tukey 算法将 N 点 DFT 分解为两个 N/2 点 DFT。

A.2 巴特沃斯极点推导

只取左半平面的 N 个极点，组成 N/2 个双二阶节。

A.3 切比雪夫多项式性质

递推：C0(x)=1, C1(x)=x, CN+1(x)=2x CN(x) - CN-1(x)。

A.4 椭圆函数性质

雅可比椭圆函数 sn(u,k) 是 sin(u) 的推广。k=0 时 sn(u,0)=sin(u)，k=1 时 sn(u,1)=tanh(u)。

A.5 频率变换细节

低通→低通：s = s/\omegac。低通→高通：s = \omegac/s。

低通→带通：s = (s2 + \omega02)/(s \cdot BW)。低通→带阻：互为倒数。

A.6 IIR 数值考虑

双二阶节级联避免系数范围过大（10阶直接型系数跨越10个数量级，级联型范围 0.01~2）。极点-零点配对使增益接近1。高 Q 节放前面。转置直接 II 型结构避免零延迟回路。

A.7 凯泽窗 beta 参数

beta=0 时为矩形窗（-13dB），beta=8 时约 -100dB。主瓣宽度近似：\Delta\omega \approx 2\pi \sqrt{1+\beta2}/N。

A.8 FFT 窗函数选择

不加窗（矩形窗）导致频谱泄露。汉宁窗是音频分析的标准选择（-32dB 旁瓣衰减，适中主瓣宽度）。

A.9 滤波器级联分析

H\text{total}(ej\omega) = \prodi=1M Hi(ej\omega)

在 dB 坐标下乘法变加法：|H\text{total}|\text{dB} = \sum |Hi|\text{dB}。

A.10 实时音频时序

128 采样点在 44.1kHz 下对应 2.9ms。前端 50ms 间隔远低于 2.9ms 实时要求，有充足裕量。

K.1 卷积的物理意义

y[n] = \sumk=-\infty\infty x[k] \cdot h[n-k]

FIR 线性相位（群延迟常数），IIR 非线性相位。贝塞尔滤波器是唯一近似恒定群延迟的 IIR。

W.1 IOperator.h 逐行解释

IOperator.h 定义了 DSP 引擎中所有"算子"（Operator）的通用接口。所谓算子，就是接受输入数据、产生输出数据的处理单元。在 MYGO-DSP 中，波形发生器和滤波器都是算子。

#pragma once
#ifndef IOPERATORH
#define IOPERATORH

第1-3行：#pragma once 和 #ifndef/#define 都是头文件保护宏，防止同一个头文件被多次包含导致编译错误。#pragma once 是编译器指令，告诉编译器"这个文件只包含一次"；#ifndef 是传统的 C/C++ 方式，通过检查宏是否已定义来防止重复包含。我们同时使用两种方式，确保最大的兼容性。

#include <cstddef>

第6行：包含 <cstddef> 标准库头文件，这个文件定义了 sizet 类型。sizet 是 C++ 中表示"大小"或"长度"的无符号整数类型，在 64 位系统中通常是 64 位无符号整数。使用 sizet 而不是 int 来表示数据长度，是因为 sizet 保证能够表示系统中任何对象的最大尺寸。

namespace mygodsp {

第8行：namespace（命名空间）是 C++ 中组织代码的机制，避免不同代码库中的同名标识符冲突。我们把所有的 DSP 代码放在 mygodsp 命名空间中。例如如果另一个库也有一个叫 IOperator 的类，使用命名空间就可以通过 mygodsp::IOperator 和 otherlib::IOperator 来区分。

class IOperator {
public:
    virtual ~IOperator() = default;
    virtual void process(const double* input, double* output, sizet numSamples) = 0;
    virtual void reset() = 0;
    virtual bool isBypassed() const = 0;
    virtual void setBypass(bool bypass) = 0;
};

第10-18行：定义一个接口类（Interface Class）。virtual 关键字表示虚函数，= 0 表示纯虚函数。一个包含纯虚函数的类就是抽象类，不能被实例化，只能作为基类使用。子类必须实现所有纯虚函数才能被实例化。

virtual ~IOperator() = default;：虚析构函数，确保通过基类指针删除派生类对象时能正确调用派生类的析构函数。= default 告诉编译器使用默认实现。

process 函数：核心处理函数。input 是指向输入数据数组的指针，output 是指向输出数据数组的指针，numSamples 是要处理的样本数量。使用指针而不是 std::vector 是为了性能——指针访问没有边界检查开销，并且可以避免不必要的内存分配。

reset 函数：重置算子的内部状态。例如对于波形发生器，重置相位累加器；对于滤波器，重置延迟线。当用户改变参数时，需要调用 reset 清除历史状态。

isBypassed 和 setBypass：旁路控制。当算子被旁路时，它应该直接将输入复制到输出，不做任何处理。这在调试和 A/B 对比时非常有用。

    int M = n / 2;
    double fc = params_.cutoff / params_.sampleRate;
    for (int i = 0; i < n; ++i) {
        int k = i - M;
        if (k == 0) {
            taps_[i] = 2.0 * fc;
        } else {
            taps_[i] = std::sin(TwoPi * fc * k) / (Pi * k);
        }
    }
}

这段代码实现理想低通滤波器的脉冲响应。公式为 h[n] = \sin(\omegac n)⁄\pi n，其中 \omegac = 2\pi fc/fs 是归一化截止角频率。当 k = 0 时，公式出现 0/0 的不定式，但实际上 \limx\to 0 \sin(x)/x = 1，所以 h[M] = 2fc/fs。

FIRFilter::process 使用直接型结构实现卷积。它维护一个环形缓冲区（delayLine），每次输入新的样本时，将它放入缓冲区，然后计算所有系数与对应延迟样本的乘积累加和。环形缓冲区的优势是避免了每次处理时移动数据的高昂开销。

W.5 IIRFilter.cpp 逐行解释

void IIRFilter::butterworthPoles(int n, std::vector<std::complex<double>>& poles) {
    poles.resize(n);
    for (int k = 0; k < n; ++k) {
        double angle = Pi * (2.0 * k + 1.0) / (2.0 * n) + Pi / 2.0;
        poles[k] = std::exp(std::complex<double>(0.0, angle));
    }
}

巴特沃斯滤波器的极点等角度分布在 s 平面左半平面的单位圆上。角度公式 \thetak = \pi⁄2 + (2k+1)\pi⁄2N 确保所有极点都在左半平面（实部小于 0），保证模拟滤波器的稳定性。k 从 0 到 N-1，共 N 个极点。

注意：这些是模拟域（s 平面）的极点。后面通过双线性变换将它们映射到数字域（z 平面）。双线性变换的公式为：

s = 2⁄T \cdot 1 - z-1⁄1 + z-1

变换后，我们需要检查所有数字极点是否在单位圆内（稳定性条件）。如果某个极点落在单位圆外，滤波器就不稳定，需要调整设计参数。

void IIRFilter::process(const double* input, double* output, sizet numSamples) {
    if (bypass_) {
        if (input != output) {
            std::copy(input, input + numSamples, output);
        }
        return;
    }
    for (sizet i = 0; i < numSamples; ++i) {
        double x = input[i];
        for (sizet j = 0; j < numBiquads; ++j) {
            double y = c.b0 * x + c.b1 * s.x1 + c.b2 * s.x2
                       - c.a1 * s.y1 - c.a2 * s.y2;
            s.x2 = s.x1; s.x1 = x;
            s.y2 = s.y1; s.y1 = y;
            x = y;
        }
        output[i] = x;
    }
}

这是 IIR 滤波器的核心处理代码。采用直接 II 型转置结构实现双二阶节。每个双二阶节的计算公式为：

y[n] = b0 x[n] + b1 x[n-1] + b2 x[n-2] - a1 y[n-1] - a2 y[n-2]

多个双二阶节级联时，前一个的输出是后一个的输入。这种级联结构对数值稳定性至关重要——如果直接实现一个高阶 IIR 滤波器，系数的量化误差会被急剧放大，可能导致滤波器不稳定或频率响应严重偏离设计值。通过分解为双二阶节，每个节的系数都在可控范围内，误差不会传播和放大。

转置结构的优点是延迟链上的数据流更规整，对 SIMD 向量化更友好，而且对定点数实现的量化误差更小。虽然在这个项目中没有使用定点数，但选择转置结构是一个良好的工程实践。

    const exePath = isProd

X.3 readline 接口

    rl = readline.createInterface({
        input: dspProcess.stdout,
        crlfDelay: Infinity,
    });

    rl.on('line', (line) => {
        try {
            const response = JSON.parse(line.trim());
            const id = response.id;
            if (id !== undefined && pending[id]) {
                if (response.status === 'error') {
                    pending[id].reject(new Error(response.data));
                } else {
                    pending[id].resolve(response.data !== undefined ? response.data : response);
                }
                delete pending[id];
            }
        } catch (e) {
            console.error('Failed to parse DSP response:', e);
        }
    });

readline.createInterface 创建了一个行读取接口，绑定到子进程的 stdout。当子进程输出一行文本后，'line' 事件触发。回调函数解析 JSON，查找对应的 pending Promise 并 resolve/reject 它。

pending 是一个对象表，存储了所有尚未收到响应的请求。每个请求在发送时，它的 resolve 和 reject 函数被存入 pending[id]。当响应到达时，用响应中的 id 查找 pending 表中的对应项，调用 resolve 或 reject 完成 Promise。如果找不到（id 不匹配或请求已超时），忽略这个响应。

这种一对一映射是确保请求和响应正确匹配的关键机制。因为 DSP 引擎是单线程的，请求和响应是严格顺序的——发送请求 1 后，在收到响应 1 之前不会发送请求 2。所以实际上 id 只是安全冗余，即使没有 id 匹配也能工作，但有了它系统更加健壮，可以检测到协议错误。

X.4 发送命令

function sendCommand(cmd, params = {}) {
    return new Promise((resolve, reject) => {
        const reqId = nextId++;
        const request = { id: reqId, cmd };
        for (const k in params) request[k] = params[k];
        pending[reqId] = { resolve, reject };
        dspProcess.stdin.write(JSON.stringify(request) + '\n');
    });
}

sendCommand 返回一个 Promise。它构造 JSON 请求，注册 Promise 回调到 pending 表，然后写入子进程的 stdin。当子进程的响应被 readline 处理后，对应的 Promise 被 resolve/reject。

注意这里使用 for (const k in params) request[k] = params[k] 而不是 {...params} 展开运算符。这是因为展开运算符会继承 params 原型的属性，而 for...in 配合 hasOwnProperty 检查更安全。不过在我们的场景中，params 总是纯粹的 Object 字面量，所以两种方式都可行。
